## What Went Well

1. **Spec review caught critical design issues early**: 4 blocking findings from 3 specialists (ADR deviation documentation, LLM extraction validation, classifier regression harness, temporal disambiguation tests). All incorporated before code, preventing expensive late-stage rework.

2. **Build checkpoint system worked smoothly**: 10 tasks, 4 checkpoints fired, 5 checkpoint-bypass decisions. Security-specialist caught error message leakage and missing typed constants (Task 6). UX-evaluator caught 32dp touch target and missing voice timeout feedback (Task 7). All REVISE findings resolved in Round 2. Zero unresolved checkpoints.

3. **Code review surfaced 4 critical blocking issues**: independent-perspective found the missing timezone field (correctness bug for ALL non-UTC users). Security found the TOCTOU race. Neither caught during build checkpoints, validating the two-layer review model (checkpoints catch approach issues; final review catches integration issues).

4. **Injectable callable pattern matured**: Now established across 4 services (Location, Photo, GoogleAuth, GoogleCalendar). Clean testing without platform channels.

5. **Sealed class pattern for ExtractionResult**: Architecture-consultant pushed for it during checkpoints. Exhaustive pattern matching was the right choice.

## What Needs Improvement

1. **Coverage dropped below 80% for the first time** (77.4%): Phase 11 added significant UI/state code without proportional test coverage. QA flagged missing SessionNotifier and VoiceOrchestrator tests as advisory during build, but non-blocking classification meant tests were never written. *Independent-perspective warns: the mechanism that caused the drop (advisory classification for missing write-path tests) is unchanged. Without reclassification, coverage will continue to erode in subsequent phases.*

2. **Education gate deferred — first Principle 6 break**: Developer explicitly tabled the education gate. This is the first deviation from a non-negotiable principle. The retro initially framed this as "monitoring," but independent-perspective correctly identifies it as a protocol break, not a deferral. The gate covers complex topics (OAuth flow, dual state machine, sealed patterns) where misunderstanding has real consequences. *Action: complete before Phase 12 begins, or formally re-defer with documented rationale per the new deferral clause (see CLAUDE.md Updates below).*

3. **Build checkpoint yield is zero blocking across 35 cumulative turns**: Reviews have 34 blocking / 125 advisory across 45 turns (0.76 yield/turn). Checkpoints have 0/2/35 (0.00). Phase 11 checkpoints drove quality improvements via 5 REVISE-then-APPROVE rounds (sealed classes, typed constants, error sanitization), but this value is not captured in yield metrics. *Independent-perspective alternative framing: checkpoints serve a categorically different function — catching issues expensive to revert mid-build — and should not be yield-compared to reviews. The right metric may be "cost of fix at checkpoint vs cost of fix at review," not finding counts.*

4. **Quality gate ran 32 times during Phase 11**: 8 formatting failures despite auto-format hook. Suggests hook timing issues or edits where hook did not trigger.

5. **Review state.json left as in_progress on session compaction**: Workflow did not complete final steps before context loss. Fixed on resume, but indicates vulnerability.

## Proposed Adjustments

1. **Add coverage-per-task tracking to /build_module**: Report delta coverage after each task. Flag immediately if it drops below threshold. Additionally, consider reclassifying missing tests for primary write paths (e.g., SessionNotifier calendar methods) as blocking rather than advisory when the 80% target is at risk.

2. **Add revise_resolved counter to protocol_yield**: Current blocking/advisory counting misses REVISE-resolved value. This makes checkpoint yield invisible in trend analysis and creates removal pressure over time. Document as known limitation in CLAUDE.md until schema is updated.

3. **Add state.json update earlier in review synthesis**: Update to complete before cleanup steps for robustness against context loss.
