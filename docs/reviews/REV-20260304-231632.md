## Request Context

- **What was requested**: Add long-press message editing to `session_detail_screen.dart` so users can correct voice transcription errors (e.g., "Shawn" vs "Sean"). After saving an edit, automatically regenerate the AI session summary so metadata stays consistent with the corrected transcript.
- **Files/scope**: `lib/database/daos/message_dao.dart`, `lib/ui/screens/session_detail_screen.dart`, `test/ui/session_detail_screen_test.dart`, `test/database/message_dao_test.dart`, `memory/bugs/regression-ledger.md`
- **Developer-stated motivation**: Voice STT frequently mishears proper nouns. Users need a way to fix transcription errors after the fact without losing the session summary.
- **Explicit constraints**: Summary must auto-regenerate after edit (user explicitly chose "always regenerate automatically"). No changes to active session flow — detail screen only.

## Blocking Findings Resolved In-Review

### B1 — Tag data-loss when metadata is null ✅ resolved
- **Severity**: High
- **Specialists**: qa-specialist + ux-evaluator (consensus)
- **Root cause**: `_regenerateSummary()` passed `null` for all tag columns to `updateSessionMetadata()` when Claude metadata was absent (offline / rule-based fallback). Drift's `update().write()` with `Value(null)` silently clobbered user-edited or AI-extracted tags.
- **Fix**: Added `metadata != null` branch — when Claude returns no metadata, the in-memory `_moodTags`/`_people`/`_topicTags` state is written back, preserving existing tags. Only when Claude returns structured metadata are the tag columns overwritten.

### B2 — Silent failure + no loading indicator for summary regeneration ✅ resolved
- **Severity**: High
- **Specialists**: ux-evaluator
- **Root cause**: `agent.generateSummary()` (a network call) had no try/catch in `_showEditMessageSheet`. Offline/timeout would throw silently. Summary header could mutate unexpectedly with no user feedback.
- **Fix**: Added `_isRegenerating` state field; sets `true` before regeneration, `false` in `finally`. Build method shows a small inline spinner+text in place of the summary header during regeneration. `on Exception` catch surfaces a `SnackBar`: "Summary could not be updated — try again later."

### B3 — Empty-string write to DB ✅ resolved
- **Severity**: Medium
- **Specialists**: qa-specialist + ux-evaluator (consensus)
- **Root cause**: Guard at line 311 did not check `edited.isEmpty`. User clearing all text and tapping Save would write `""` to `updateMessageContent`, producing a blank bubble.
- **Fix**: Guard now reads: `if (edited == null || edited.isEmpty || edited == message.content || !mounted) return;`

## Advisory Findings (open)

### A1 — Accessibility: no Semantics on long-press GestureDetector (Medium)
- `GestureDetector` has no `Semantics` wrapper. TalkBack cannot announce "double-tap and hold to edit." Screen reader users have no path to this feature.

### A2 — Discoverability: no visual affordance for long-press (Medium)
- No tooltip, edit pencil icon, or hint text on user bubbles. Voice-first ADHD users who never try long-press may not discover the feature.

### A3 — Cancel path and whitespace-only save untested (Low)
- No tests for: cancel button path (no DB write), whitespace-only input (now correctly rejected by B3 fix but still untested).

### A4 — Sheet max height unconstrained on long messages (Low)
- `Column(mainAxisSize: MainAxisSize.min)` with no `maxHeight` cap. On very long pre-filled content the sheet could cover the entire screen.

### A5 — Missing `@Tags(['regression'])` annotation on regression test (Low)
- The first test in the `'Message editing'` group includes `'(regression)'` in its name but lacks the `@Tags(['regression'])` annotation required by `testing_requirements.md`. A comment referencing the ledger entry was added as partial mitigation.
