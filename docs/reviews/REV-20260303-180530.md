## Summary

Phase 1 Task 10 adds dedicated `CheckInScreen` (slider-based, no chat chrome), `CheckInHistoryScreen`
(expandable cards with score chip and per-question answer bars), `completeCheckInSession()` (bypasses
the empty-session auto-discard guard), `watchAllResponsesWithAnswers()` (IN-clause N+1-free stream),
and export completeness for check-in data. ADHD clinical UX constraints are fully upheld — no gap-shaming,
no streaks, neutral score chip palette. Four blocking findings: one data-correctness issue (history uses
active-only item filter) and three accessibility gaps (missing `Semantics` annotations).

## Findings by Specialist

### Architecture Consultant
- **F1 (Medium — BLOCKING)**: `checkInHistoryProvider` calls `getActiveItemsForTemplate()` (filters `isActive=true`). Deactivated questionnaire items silently lose question text in history view — `'Question N'` shown instead. ADR-0032 supports item deactivation. Fix: add `getAllItemsForTemplate()` (no `isActive` filter) to `QuestionnaireDao` for the history path.
- **F2 (Medium — Advisory)**: `async*` + `await-for` + `asyncMap` pattern is novel relative to all other `StreamProvider` usages — undocumented. Add inline comment explaining the approach and backpressure characteristic.
- **F3 (Low — Advisory)**: `_sessionComplete=true` set inside `build()` — only screen in codebase to mutate widget-local state in the build phase. Correct but inconsistent; consider `ref.listen` in `initState` for future refactor.
- **F4 (Low — Advisory)**: `'pulse_check_in'` string literal duplicated across routing and mode picker. Define a `JournalingMode` constants class (not enum, per ADR-0025).
- **F5 (Informational)**: Score chip `toStringAsFixed(0)` floating-point edge case — handled correctly by rounding; no change required.

## Required Changes Before Merge

1. **B1 — Add `getAllItemsForTemplate()` to `QuestionnaireDao`** and use it in `checkInHistoryProvider` instead of `getActiveItemsForTemplate()`. The history view must show question text for all items a user ever answered, including those now deactivated.
2. **B2 — Add `Semantics` to `_CheckInEntryCard` `InkWell`** (or replace with `ExpansionTile`): `Semantics(button: true, label: '$date, score X, ${_expanded ? 'expanded' : 'collapsed'}')`.
3. **B3 — Add `Semantics` to `_AnswerRow` progress bar row**: `Semantics(label: '$question: $value', excludeSemantics: true)` wrapping the `Row`.
4. **B4 — Add `semanticFormatterCallback` to `Slider` in `pulse_check_in_widget.dart`**: `(v) => '${v.round()} out of ${scaleMax.toInt()}'`.
