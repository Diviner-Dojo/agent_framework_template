## Recommended Improvements (advisory)

**A1 (QA, Medium): "Set the call" false positive edge case**
`call` is in the event noun list. "Set the call to silent" (5 words, bypasses short-message guard) matches `^(add|set)\b.{0,15}\bcall\b` → calendarScore = 0.5 → calendar card shown. This is a pre-existing risk for "add" now extended to "set". The confirmation gate (ADR-0020 §8) prevents any destructive auto-creation; the user dismisses the card. "Set a call" is predominantly a scheduling idiom in everyday speech. Advisory: add a documented test pinning the accepted behavior (e.g., `'"Set the call to silent" shows calendar card (acceptable ambiguity — guarded by confirmation gate)'`).

**A2 (QA, Low): Confidence assertion could be tightened**
Both new tests assert `greaterThanOrEqualTo(0.5)`. The scoring path for these inputs is fully deterministic (one `_calendarIntentPattern` match, no temporal boost → 0.5 exactly). Optionally tighten to `equals(0.5)` to detect silent score inflation. Not blocking.

**A3 (Independent, High recurrence likelihood): Three divergent event noun lists**
The event noun list appears independently in three locations:
- `_calendarIntentPattern` (lines 439–440): `meeting|appointment|event|dinner|lunch|call|reservation`
- `_hasStrongCalendarSignal` (line 544): same 7 nouns
- `_eventNounPattern` (~line 459): 15 nouns — includes `breakfast|brunch|date|party|interview|conference|hangout` not in the others

This divergence is the structural root cause that produced BOTH PR #56 and PR #57. Sprint N+1 priority: extract to `static const _calendarEventNouns = r'(meeting|...|reservation)'` shared by all three. Estimated effort: <30 minutes, zero behavior change.

**A4 (Independent, High recurrence likelihood): Manual sync invariant between two locations**
`_calendarIntentPattern` and `_hasStrongCalendarSignal` must be kept in sync by convention — no code enforces the invariant. Every future verb or noun addition risks the same regression class (fail to update one of the two). Addressed by A3's shared-constant refactor.

**A5 (Independent, Pre-existing): "set a reminder" routes to journal**
"set a reminder" (4 words) — "reminder" is not in the event noun list, so `_hasStrongCalendarSignal` returns false → 4-word guard fires → journal. But "set a reminder" is naturally a reminder intent. `_hasStrongReminderSignal` only matches `^remind\b`. Newly visible due to this PR's expansion of "set". Separate follow-up recommended.
