## Request Context

- **What was requested**: Implement Deepgram Nova-3 as primary STT engine per ADR-0031 — new `DeepgramSttService implements SpeechRecognitionService`, new `deepgram-proxy` Supabase Edge Function WebSocket relay, provider wiring, settings UI, and tests.
- **Files/scope**: Edge Function proxy, Dart service, providers, environment config, test suite, settings screen
- **Developer-stated motivation**: Replace Android SpeechRecognizer (15–20% WER, ~800ms endpoint detection) with Deepgram Nova-3 (6–9% WER, journaling-tuned 2000ms endpoint detection) per ADR-0031 ADHD voice capture reliability requirements.
- **Explicit constraints**: ADR-0005 proxy pattern — no Deepgram API key in client code; key lives in Supabase secrets only.

## Required Changes (Blocking — All Resolved)

### B-1: Test duplication — tests verified test-double, not production code
**Raised by**: architecture-consultant, qa-specialist
**Status**: RESOLVED

`_TestableDeepgramSttService._parseAndCollect()` duplicated the full ADR-0031 parsing state machine. Tests were verifying the test double, not the production `_handleResultsEvent` / `_handleUtteranceEndEvent` methods.

**Fix applied**:
- Removed class-level `coverage:ignore-start/end`
- Added narrow `coverage:ignore` blocks on `_connectAndCapture()`, `stopListening()`, `dispose()` only
- Parsing methods (`_onSocketMessage`, `_handleResultsEvent`, `_handleUtteranceEndEvent`) now outside coverage:ignore
- Added `@visibleForTesting initStreamForTesting()` — creates `StreamController.broadcast(sync: true)`, returns it for test subscription
- Added `@visibleForTesting injectMessageForTesting(String)` — delegates to production `_onSocketMessage()`
- Test subclass rewritten to use these hooks; all 10 parsing tests now exercise production code

### B-2: Double-`startListening()` guard untested
**Raised by**: qa-specialist
**Status**: RESOLVED

The `if (_isListening) throw StateError('Already listening')` guard in `startListening()` had no test.

**Fix applied**: Added lifecycle test `'startListening throws StateError if already listening'` — verifies the guard fires synchronously before `_connectAndCapture()` is reached.

### B-3: API key in Edge Function URL / coverage:ignore scope too broad
**Raised by**: security-specialist
**Status**: RESOLVED

The `deepgramUrl` string containing `?access_token=${key}` could appear in logs if any log statement received the URL variable. Class-level `coverage:ignore` also concealed auth header construction in `_connectAndCapture()`.

**Fix applied**:
- Added 3-point comment explaining why `access_token` query param is safe server-side: (1) key never transmitted to client; (2) `deepgramUrl` is never logged; (3) logs only accessible to project admin
- Added note to migrate to Authorization header if Deno adds stable WebSocket header support
- Narrow coverage:ignore blocks replace class-level ignore (auth header construction in `_connectAndCapture()` remains ignored as it requires real networking)

### B-4 (A-2 from arch+perf): Unbounded `pendingAudio` queue in Edge Function
**Raised by**: architecture-consultant, performance-analyst
**Status**: RESOLVED

The pending audio queue before Deepgram WebSocket opens had no size cap — potential memory exhaustion under slow connections or Deepgram cold starts.

**Fix applied**: Added `MAX_PENDING_AUDIO = 100` constant (≈2s of audio at 20ms/chunk). Oldest-first eviction in `clientSocket.onmessage`.
