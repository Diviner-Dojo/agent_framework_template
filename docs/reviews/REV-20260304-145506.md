## Advisory Findings

### A1 — capturePhotoDescription non-paused branch untested (Medium)
**Specialist**: qa-specialist
The wasInContinuousMode path and the else path in the new conditional have no capturePhotoDescription-specific tests. A future refactor could silently break the normal (non-paused) flow.
**Recommendation**: Add test for non-paused continuous case (caller did not pause() before capturePhotoDescription).

### A2 — pulse_check_in dispatch branch not covered (Medium)
**Specialist**: qa-specialist
Only 3 of 5 palette dispatch branches are tested. The pulse_check_in branch routes to /check_in — a distinct navigation path from /session that should be pinned.
**Recommendation**: Add testWidgets for Check-In tile verifying /check_in route.

### A3 — Post-startContinuousMode assertion timing concern (Low)
**Specialist**: qa-specialist
The Bug 2 regression test asserts phase==listening immediately after startContinuousMode without a microtask yield. Relies on mockTts.autoComplete=true. If mock setup changes, the incidental assertion becomes flaky.
**Recommendation**: Move/remove the incidental pre-pause assertion or add a short await.

### A4 — Silence timeout path in capturePhotoDescription untested (Low)
**Specialist**: qa-specialist
No test for capturePhotoDescription returning null (silence timeout path) with paused-state correctly restored.
**Recommendation**: Acknowledge as known gap or make timeout injectable.

### A5 — maxLines: 6 not adaptive; minLines absent (Medium)
**Specialist**: ux-evaluator
On 360dp devices with voice mode controls active, 6 lines can consume 120-150dp leaving little space for the message list. minLines is absent — multi-line affordance is invisible until first line wrap.
**Recommendation**: Add minLines: 1. Consider maxLines: 4 or ConstrainedBox wrapper.

### A6 — Missing textInputAction: TextInputAction.send (Low)
**Specialist**: ux-evaluator
With maxLines > 1, Flutter's default textInputAction is newline. Enter inserts a newline instead of submitting. onSubmitted callback exists but never fires via keyboard. Pre-existing, now more noticeable.
**Recommendation**: Add textInputAction: TextInputAction.send.

### A7 — FAB not visually disabled during _isStarting (Low)
**Specialist**: ux-evaluator
FAB with onPressed: null does not visually dim in Material 3 by default. Spinner communicates loading, but color/elevation unchanged. Tooltip only surfaces on long-press.
**Recommendation**: Apply dimmed backgroundColor when _isStarting, or add Semantics label.

### A8 — Input container uses viewPadding.bottom (pre-existing) (Low)
**Specialist**: ux-evaluator
viewPadding.bottom doesn't collapse when keyboard opens (unlike padding.bottom). With resizeToAvoidBottomInset: true on the Scaffold, the inset is double-counted. Steals ~24-34dp from message list when keyboard is active. Pre-existing, interacts with maxLines fix.
**Recommendation**: Use MediaQuery.of(context).padding.bottom in the input container.
