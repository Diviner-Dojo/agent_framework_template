## Advisory Findings (Non-Blocking)

### A-1 [Medium, QA] — Missing widget test for text-primary mode `textInputAction.send`

**File**: `test/ui/journal_session_screen_test.dart` (gap)

The A8 regression test update asserts `TextInputAction.newline` in the default voice+text mode but the text-primary mode (`TextInputAction.send`) path has no widget test. The original A6 regression protected unconditional `.send`; A8 split this into a conditional. If the conditional were accidentally inverted, the existing test would still pass because the default mode is `false`.

**Recommended**: Add a companion test that taps the `'Text'` segment button to activate `_isTextInputMode`, then asserts `textField.textInputAction == TextInputAction.send`.

### A-3 [High, UX] — No discoverability affordance in voice+text mode for Enter-to-newline change

**File**: `lib/ui/screens/journal_session_screen.dart:748`

In voice+text mode, `TextInputAction.newline` suppresses `onSubmitted` on Android — pressing Enter inserts a newline, not a submit. There is no in-field signal indicating Enter won't submit. Users habituated to Enter-to-submit will silently get a newline with no pointer to the send icon. The send icon is always reachable via `_buildActionButton`'s default case — no dead end, but discoverability gap is real.

**Recommended**: Add `helperText: 'Tap send icon to submit'` (visible in voice+text mode when mic is inactive) to `InputDecoration`. Advisory for next sprint.

---

### A-4 [Medium, UX] — Y-axis labels (0.0/0.5/1.0) contradict plain-language subtitle

**File**: `lib/ui/screens/check_in_history_screen.dart:804–811`

The A9 subtitle ("Chart scaled to your own range — top of chart is your personal highest") is correct and ADHD-compliant. However, Y-axis tick labels still display `0.0 / 0.5 / 1.0`, referencing the 0–1 normalization scale. A user reading the subtitle then looking at the axis sees "1.0" where "your personal highest" was promised. This is a pre-existing inconsistency exposed by the subtitle improvement; axis labels should become `Low / Mid / High`.

**Recommended**: Follow-up advisory; out of scope for this sprint.

---

### A-5 [Low, QA] — No mixed-mode provider test

**File**: `test/providers/notification_providers_test.dart`

No test exercises both `failedTaskIds` and `rescheduled` loops in a single provider invocation. A shared-state bug between loops would be missed. Add a fourth test with two tasks (one rescheduled, one failed) using a mixed-result scheduler.

**Recommended**: Add before next sprint.
