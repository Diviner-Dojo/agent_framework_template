## Summary

Phase 7A Voice Foundation adds push-to-talk voice input to the journal app: 7 new files under `lib/`, 4 modified files, 1 ADR. The architecture is clean with well-defined module boundaries, correct abstract/concrete separation, and consistent platform channel patterns. Five blocking findings must be addressed before merge: empty SHA-256 checksums on model downloads (supply chain risk), a `wasVoiceInput` logic bug that misclassifies all voice messages as TEXT, TTS never initialized before use, missing loading indicator during 5-8s STT model load, and missing accessibility tooltips on action buttons. Fifteen advisory findings cover testing gaps, UX polish, and minor code quality issues.

## Required Changes Before Merge

1. **Populate SHA-256 checksums and implement verification** (security-specialist #1): Retrieve real checksums from HuggingFace API, populate `ModelFileInfo.sha256` fields, implement streaming SHA-256 verification using `crypto` package after each file download. This is a supply chain integrity requirement — ONNX files are loaded directly into the native runtime.

2. **Fix wasVoiceInput tracking** (qa-specialist #1, ux-evaluator #8): Replace `final wasVoiceInput = _isRecording` with a separate `_lastInputWasVoice` flag set in the stream listener when `isFinal` triggers auto-stop, consumed and cleared in `_sendMessage()`. Add a test verifying voice messages are stored with `inputMethod='VOICE'`.

3. **Initialize TTS before first use** (ux-evaluator #2, qa-specialist #5): Add lazy initialization in `_speakAssistantMessage()`: check if service needs init, call `initialize()` before `speak()`. Without this, TTS silently fails on every voice session.

4. **Add loading indicator during STT initialization** (ux-evaluator #1): The 5-8 second `sttService.initialize()` call has no UI feedback. Add an `_isInitializing` state that shows a spinner in the action button area while the model loads.

5. **Add accessibility tooltips to action buttons** (ux-evaluator #6): Add `tooltip: 'Start voice input'` / `'Stop recording'` / `'Send message'` to all three `IconButton.filled` variants. Screen readers currently announce bare "Button" with no context.
