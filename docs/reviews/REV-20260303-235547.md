## Required Changes Before Merge

**B-1 (RESOLVED IN-REVIEW)**: A8 `Switch.onChanged` lacked try/catch on `dao.updateItem()`. A failed write left `enabled` parameter in scope, causing `'Question deactivated.'` SnackBar to appear even when the question was not actually deactivated — false confirmation of a failed operation. Fixed by wrapping the write in `try/catch`, showing `'Could not update question. Try again.'` on exception, and `return`-ing early before the Undo SnackBar. All 19 tests pass with fix applied.
