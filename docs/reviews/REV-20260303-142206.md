## Summary

Phase 1 (Pulse Check-In questionnaire — 6 items, composite score, text slider + voice mode) and
Phase 2A/B (gap-shaming removal across all three conversation layers, non-shaming recovery banner)
across 15 production files. The core questionnaire architecture is sound — clean state ownership,
atomic saves, N+1 avoidance, thorough score computation. However 6 blocking findings prevent merge:
the Phase 2B recovery banner violates two CLAUDE.md clinical UX constraints (implicit gap-shaming,
dismissal re-escalation), the numeric parsing adapter duplicates NumericParserService with reduced
capability causing voice/text behavioral divergence, Phase 2A regression tests are missing a tag
and ledger entry, and ADR-0032 is in 'proposed' status describing a materially different schema
than the implementation. Fixes for B1–B4 are straightforward (1–5 lines each); B5 (ADR update)
is documentation; B6 (banner tests) should wait for the B1/B2 design resolution.

## Required Changes Before Merge

**B1 — Recovery Banner: Remove Gap-Detection Trigger**
`lib/ui/screens/session_list_screen.dart:74-79`
The `daysSinceLast >= 3` trigger violates CLAUDE.md "not even implicitly." Two compliant options:
- Option A: Show banner universally on every session (remove daysSinceLast check, keep CTA)
- Option B: Remove banner entirely; Quick Check-In already accessible via FAB long-press
Must choose one before B2 dismissal fix and B6 tests.

**B2 — Recovery Banner Dismissal: Persist Beyond Widget Lifecycle**
`lib/ui/screens/session_list_screen.dart:37`
Replace `bool _isRecoveryBannerDismissed = false;` with app-level Riverpod `StateProvider<bool>`
or SharedPreferences date key. Widget-local state resets on navigation, violating "must not
escalate after dismissal."

**B3 — Replace _NumericParserAdapter with NumericParserService**
`lib/providers/questionnaire_providers.dart:422-477`
Replace the 53-line private adapter with:
```
final _numericParserProvider = Provider((_) => const NumericParserService());
```
Import `numeric_parser_service.dart` (no circular dependency). Update `handleVoiceAnswer()` to
call `parser.parse(rawText, scaleMin: ..., scaleMax: ...)` directly.

**B4 — Tag LLM-Layer Regression Tests + Add Ledger Entry**
`test/layers/local_llm_layer_test.dart` — add `@Tags(['regression'])` library annotation.
`memory/bugs/regression-ledger.md` — add Phase 2A gap-shaming removal entry:
  - Bug: daysSinceLast injected into AI context hints and used for rule-based greeting
  - Root cause: ADHD UX constraint missing from original implementation
  - Fix: Removed from claude_api_layer.dart, rule_based_layer.dart, local_llm_layer.dart
  - Regression tests: test/layers/gap_shaming_removal_test.dart, test/layers/local_llm_layer_test.dart

**B5 — Update ADR-0032 to Accepted with As-Built Schema**
`docs/adr/ADR-0032-pulse-checkin-questionnaire-schema.md`
Change status 'proposed' → 'accepted'. Replace schema definition with actual tables.dart
definitions (QuestionnaireTemplates, QuestionnaireItems, CheckInResponses, CheckInAnswers).
Document deferred user_checkin_config (Phase 1 Task 8, schema v11). Per CLAUDE.md Principle 1.

**B6 — Add Recovery Banner Widget Tests**
`test/ui/session_list_screen_test.dart` — add after B1/B2 design fix:
- Banner shown when most recent session >= 3 days ago (or universally if Option A chosen)
- Dismissed by close button — does not reappear
- Dismissed by "Just browse" — does not reappear
- Not shown within same session after dismissal

## Strengths

- **Clean state boundary**: CheckInNotifier fully self-contained, no SessionNotifier extension.
  Coordination via sessionId matches established pattern of task_providers and calendar_providers.
- **Atomic save**: `saveCheckInResponse` wraps response+answers in `db.transaction()`. No orphaned
  partial records on failure.
- **N+1 avoidance**: `getAllResponsesForSession` uses single IN-clause query with index awareness.
- **Score service purity**: `CheckInScoreService` is const, no async, no side effects. Explicit
  INVARIANT comment documents reverse-scoring formula correctly (scaleMax + scaleMin - raw, not +1).
- **Phase 2A thoroughness**: Gap-shaming removal consistent across all three conversation layers
  with explicit comments documenting the ADHD UX motivation.
- **System default protection**: `deleteTemplate()` no-ops for isSystemDefault=true; historical
  response FK integrity preserved.
- **ADHD copy compliance**: "Saved. That's enough." closing text, no evaluative language, no
  duration references in any displayed copy.
- **Test quality**: Acceptance criteria test (closeTo(68.5, 0.2)), idempotency 3-variant coverage,
  real DAOs with in-memory DB over-mocking, meaningful all-skipped assertion.
