## Summary

Phase 10: Location Awareness adds GPS location capture to journal sessions with a privacy-first design (ADR-0019). The implementation is architecturally sound with strong patterns (injectable callables, fire-and-forget capture, rounding at acquisition) and a well-defined sync boundary (name-only to cloud). Four blocking findings require attention before merge: an over-declared Android permission, a non-executable sync boundary test, missing loading feedback on clear operation, and a UX gap in permission request timing. Twelve advisory findings address accessibility, test coverage gaps, and privacy disclosure refinements.

## Required Changes Before Merge

1. **B1: Remove `ACCESS_FINE_LOCATION` from AndroidManifest.xml** until the "High Precision" settings toggle is implemented. `ACCESS_COARSE_LOCATION` alone is sufficient for the current feature. The geolocator package works with coarse permission.

2. **B2: Make sync boundary test executable.** Extract a `@visibleForTesting` method `buildSessionUpsertMap(JournalSession, String)` from `uploadSession()` in `sync_repository.dart`. Test that the returned map contains `location_name` and does NOT contain `latitude`, `longitude`, or `location_accuracy` keys. This fulfills the ADR-0019 ┬¦3 promise of a "load-bearing" regression test.

3. **B3: Add loading state for Clear Location Data.** Add a local `_isClearingLocation` bool. Set true before `clearAllLocationData()`, false in `finally`. Disable the button while clearing. Prevents double-tap and eliminates the silence gap.

4. **B4: Request location permission at toggle-on time.** Add a `checkAndRequestPermission()` helper to `LocationService` that returns `bool`. Call it from the toggle's `onChanged` handler. If denied, show SnackBar with guidance and don't enable the toggle. If `deniedForever`, direct to `Geolocator.openAppSettings()`. This keeps the system permission prompt contextually linked to the user's deliberate action.

## Recommended Improvements (Non-Blocking)

1. **A1/A6**: Add `Semantics(label: 'Location: ${session.locationName}')` to session card location icon. Consider adding location name text alongside icon.
2. **A2b**: Lift privacy disclosure to `bodySmall` at `onSurface` color, or wrap in a tinted container for visual weight.
3. **A3b**: Conditionally disable Clear Location Data button when location toggle is off.
4. **A4**: Fix `_captureLocationAsync` doc comment ŌĆö remove mismatched first sentence about sync.
5. **A5**: Consider a 30-60 minute staleness threshold for `getLastKnownPosition`. Or document the intentional acceptance of stale positions.
6. **A6b**: SharedPreferences backup restore risk ŌĆö consider excluding `location_enabled` from Android Auto Backup XML rules.
7. **A7**: Add assertion for `Icons.location_on_outlined` in session list test.
8. **A8**: Add location chip tests to `session_detail_screen_test.dart`.
9. **A9**: Add interaction tests for location toggle and clear data in settings screen.
10. **A10**: Strengthen geocoding disclosure to mention Google account correlation.
11. **A11**: Add comment to `_defaultGetCurrentPosition` explaining `LocationSettings` choice.
12. **A12**: Improve clear dialog copy to explain when cloud sync happens.
