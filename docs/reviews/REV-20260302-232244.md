## Required Changes (blocking)

**B1 — RESOLVED during review: plan.md synthesis step lacked `## Request Context` requirement.**
`plan.md` added the context-brief Step 3.5 but did not update the synthesis step to require a `## Request Context` opening section — a hybrid state where context-brief is captured at turn_id=1 but not referenced in the synthesis structure. QA (F7) and docs-knowledge (F1) independently identified this. Fixed: `plan.md` Step 5 now requires `## Request Context` as the opening section of the synthesis event, consistent with `review.md`.

**B2 — RESOLVED during review: Retro standing agenda appended after Step 7 (structurally weak placement).**
The standing agenda was a free-floating block after Step 7 rather than a numbered step embedded in the workflow sequence. Architecture-consultant (F2) identified that agents completing the numbered workflow reach 'Step 7: Present' and may stop, missing the gate entirely. Fixed: restructured as **Step 5.5** (before finalization), with a numbered heading integrated into the sequence before Step 6.

**B3 — RESOLVED during review: Retro disposition condition always fires (dead code).**
The condition `findings.disposition != 'open'` returns zero rows structurally — the disposition column is not yet implemented, so the "all are false" branch was permanently unreachable. QA (F2, elevated to blocking in Round 2) identified this. Fixed: rewritten as three explicit observable signals (Signal A: specialist echo, Signal B: framing drift, Signal C: disposition activity with "pending implementation" note) with a clear decision rule.

## Recommended Improvements (advisory)

**A1 (QA + Architecture, Medium): turn_id=1 invariant is convention-only, not tooling-enforced**
The turn_id=1 guarantee for context-brief is enforced by command workflow order only. Any future reorder silently breaks extraction pipeline integrity without error. Short-term: add an `# INVARIANT: This must be the first write_event.py call` comment block in each command's context-brief step. Long-term: `write_event.py --assert-first-turn` flag.

**A2 (QA + Architecture, Medium): Cold-start edge case — no fallback for commands invoked without session context**
If `/review` (or other commands) is invoked fresh with no prior conversation, the context-brief step has no fallback instruction. Architecture frames this as a contract gap: add a guard instruction — "If no conversational context exists prior to this command, populate all four fields as '(none stated)' and add tag `context-brief-cold-start`." Advisory: add fallback guidance in a follow-up pass.

**A3 (Docs, Medium): CLAUDE.md doesn't enumerate which commands emit context-brief or why exclusions apply**
The capture pipeline section doesn't list which commands emit context-brief events. Developers can't determine from CLAUDE.md whether `/deliberate`, `/retro`, `/build_module`, `/plan` participate, or why `/analyze-project` and `/meta-review` are excluded. Recommend adding: "Context-brief events (turn_id=1) are emitted by: /review, /deliberate, /build_module, /plan, /retro. Excluded: /analyze-project (outward-facing scouting, no developer request context), /meta-review (aggregate analysis, no single request)."

**A4 (Docs, Low): build_module.md missing privacy filter sentence**
The instruction "Strip business context (deadlines, client names, regulatory pressures) — record structural intent only" appears in `review.md` and `deliberate.md` context-brief steps but was omitted from `build_module.md`. Advisory: add in a follow-up consistency pass.

**A5 (Docs + Architecture, Low): ADR-0030 'deferred' wording — create stub file**
`CLAUDE.md` references "ADR-0030 deferred" but no stub file exists in `docs/adr/`. "Deferred" implies a decision was started and paused; a placeholder stub with `status: proposed` and a one-line problem statement would make the reference resolvable and prevent confusion.

**A6 (Docs, Low): documentation_policy.md rule has no enforcement mechanism noted**
The new rule "All facilitator synthesis events must include a `## Request Context` section" has no automated check — unlike the context-brief requirement (enforced by command ordering) or regression test requirement (enforced by `quality_gate.py`). Recommend adding a parenthetical: "(Verified by review facilitator during synthesis step — no automated check.)"
