## Summary

Three device-testing fixes reviewed: (1) `AndroidAudioSource.voiceRecognition` + `manageBluetooth:false` to fix Deepgram STT silence on Samsung Galaxy S21 Ultra after TTS playback; (2) `ttsReleaseDelay` increased to 500ms for Android audio focus release; (3) `_regenerateSummary()` now passes USER-only messages to prevent stale ASSISTANT content corrupting name extraction after transcript edits. Two blocking findings resolved in-review: regression test added for the USER-only filter, and `ttsReleaseDelay` made Android-conditional.

## Request Context

- **What was requested**: Fix Deepgram STT not transcribing on Samsung Galaxy S21 Ultra after ElevenLabs TTS playback; fix USER-only messages for metadata regeneration after message edit; increase TTS release delay for audio routing
- **Files/scope**: lib/services/deepgram_stt_service.dart, lib/providers/voice_providers.dart, lib/ui/screens/session_detail_screen.dart
- **Developer-stated motivation**: Three device testing bugs: (1) Deepgram STT listens but never transcribes on Samsung One UI after TTS — 15-second silence timeout fires; (2) metadata extraction uses stale ASSISTANT messages (e.g., "Shawn") after user corrects a transcript; (3) audio focus release timing was too aggressive for Samsung audio routing
- **Explicit constraints**: Must not break emulator test; must not require proxy/auth changes; Samsung S21 Ultra (Android 14, One UI) specific audio routing behavior

### Performance Analyst
- **B2 (Blocking — RESOLVED)**: `ttsReleaseDelay: const Duration(milliseconds: 500)` fires unconditionally on all platforms. Fixed: `Platform.isAndroid ? const Duration(milliseconds: 500) : Duration.zero`. iOS gets no delay penalty.
- **A4 (Advisory — Open)**: Double `getMessagesForSession` calls in `_regenerateSummary`. Negligible at current scale.
- **A5 (Advisory — Open)**: Disabled noise suppression sends raw audio upstream. Accepted as intentional per ADR-0031 (Deepgram handles server-side).
- **A6 (Advisory — Open)**: `stopListening` 500ms flush + `ttsReleaseDelay` 500ms = ~1000ms dead time per turn on Android. Acceptable for journaling cadence.
