## Summary

Phase 8A extracts the ConversationLayer strategy pattern from AgentRepository, adds the layer selection/fallback infrastructure, journal-only mode, and settings UI toggles. The refactoring is clean, well-documented (ADR-0017), and preserves all existing behavior (715 tests pass, 80.6% coverage). Two blocking issues identified: incomplete exception handling in the fallback chain and missing UX feedback on a disabled toggle.

## Findings by Specialist

- **MEDIUM**: `_generateFallbackSummary` duplicated between ClaudeApiLayer and RuleBasedLayer (DRY violation)
- **MEDIUM**: `localLlmLayer` public mutable field with no encapsulation
- **MEDIUM**: `agentRepositoryProvider` rebuild risk mid-session when preferences change
- **LOW**: `journalOnlyMode` uses public field while `_preferClaude` uses private+setter (inconsistent)
- **INFO**: Consider `abstract interface class` for ConversationLayer

## Required Changes Before Merge

1. **Add generic Exception catch to fallback chain** (architecture-consultant finding 1): Add `on Exception` as a second catch clause after `on ClaudeApiException` in all four dispatch methods (`getGreeting`, `getFollowUp`, `generateSummary`, `getResumeGreeting`) in `agent_repository.dart`. This ensures any `ConversationLayer` failure — including future `LocalLlmLayer` exceptions — falls back to `RuleBasedLayer` per ADR-0017 Section 4.

2. **Add conditional subtitle to disabled Prefer Claude toggle** (ux-evaluator finding 1): When `journalOnly` is true, change the subtitle of the "Prefer Claude when online" `SwitchListTile` to explain why it is disabled (e.g., "Disabled while Journal only mode is on"). Users currently see a grayed-out toggle with no explanation.

## Recommended Improvements (Non-Blocking)

1. Extract `_generateFallbackSummary` to shared utility to eliminate DRY violation between layers
2. Replace `localLlmLayer` public field with setter method for Phase 8B control point
3. Apply same setter pattern to `journalOnlyMode` for consistency with `_preferClaude`
4. Add missing tests: getResumeGreeting fallback, preferClaude OFF + LLM + Claude, Claude null summary
5. Remove placeholder elements from AI Assistant card (reintroduce in Phase 8B with real functionality)
6. Rename "AI Assistant" card to "AI Behavior" or "Conversation Mode" to avoid collision with "Digital Assistant"
7. Add `ExcludeSemantics` wrapper on decorative `smart_toy_outlined` icon
8. Consider swapping toggle order so "Journal only mode" (parent) appears above "Prefer Claude" (child)
9. Fix async throw assertions to use `expectLater` pattern in claude_api_layer_test.dart
