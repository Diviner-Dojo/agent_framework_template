## Summary

This review covers four bug fixes to the journal-only voice mode: the orchestrator stuck-in-processing bug (`acknowledgeNoResponse()`), the "goodbye" dead-code bug (`shouldEndSession()` reorder), advisory improvements from the previous review (A1/A4/A6 cross-reference comments, `isContinuousMode` guard), and a new back-button bug fix (`_endSessionAndPop()` no longer force-pops on success). One blocking finding was identified: the back-button fix introduced a regression where the "Done" button in the AppBar remains visible and silently no-ops during the closing summary state (`isClosingComplete=true`). All other findings are advisory.

## Required Changes Before Merge

1. **[B1] Hide or replace Done button during closing summary state** ŌĆö When `isClosingComplete=true`, the Done button condition (`!isSessionEnding && activeSessionId != null`) evaluates to true, rendering a tappable button that silently no-ops via the `isSessionEnding` re-entry guard in `endSession()`. Add `!sessionState.isClosingComplete` to the Done button visibility condition (`journal_session_screen.dart:368`). Optionally replace the Done button during `isClosingComplete` with a "Close" `TextButton` calling `_dismissAndPop()` to make the correct affordance explicit.

## Recommended Improvements (Non-Blocking)

1. **[A1] AppBar title during closing summary state** ŌĆö Add `isClosingComplete ? 'Entry Saved' : 'Journal Entry'` to the AppBar title. One-line change; makes the two-step exit flow feel intentional to users (`journal_session_screen.dart:348`). No ADR required.

2. **[A2] Test PTT branch of `acknowledgeNoResponse()`** ŌĆö Add test with orchestrator in push-to-talk+processing state; call `acknowledgeNoResponse()`; assert `state.phase == VoiceLoopPhase.idle` and audio focus abandoned (`voice_session_orchestrator_test.dart`).

3. **[A3] Test `_resumeOrchestratorIfVoiceMode()` with voice enabled** ŌĆö Add a provider-level test overriding `voiceModeEnabledProvider=true` with a wired orchestrator to verify `acknowledgeNoResponse()` is invoked after journal-only or handled-intent messages (`session_providers_test.dart`).

4. **[A4] Document `onEndSession` auto-dismiss as intentional** ŌĆö Add inline comment at `journal_session_screen.dart:158` explaining why voice-initiated endings auto-dismiss without showing the closing summary (voice users get TTS summary; manual dismiss would be redundant) (`journal_session_screen.dart:158-162`).

5. **[A5] Update ADR-0016 ┬¦1 for new communication channel** ŌĆö Add addendum documenting that `SessionNotifier` may call `acknowledgeNoResponse()` directly for no-response scenarios, alongside the existing callback contract (`docs/adr/ADR-0016-*.md`).

6. **[A6] Replace `debugPrint` with `AppLogger.e()`** ŌĆö Replace `debugPrint('[JournalSession] endSession failed: $e')` with `AppLogger.e('session', 'endSession failed during back-navigation: $e')` at `journal_session_screen.dart:1188`.

7. **[A7] Update orchestrator state machine comment** ŌĆö Add `processing ŌöĆŌöĆ[no response, continuous]ŌöĆŌöĆ> listening` and `processing ŌöĆŌöĆ[no response, PTT]ŌöĆŌöĆ> idle` to the state machine transitions at `voice_session_orchestrator.dart:9-18`.

8. **[A8] Add `@Tags(['regression'])` to new orchestrator tests** ŌĆö Tag `'acknowledgeNoResponse() is a no-op when not in processing phase'` and `'acknowledgeNoResponse() transitions from processing to listening (regression)'` per `testing_requirements.md` (`voice_session_orchestrator_test.dart`).

## Strengths

- Regression ledger entries are complete and accurate for all three fixed bugs, with correct test file references
- Bug 3 regression test uses `isClosingComplete == true` (not `activeSessionId == null`) with a clear explanatory comment ŌĆö demonstrates real understanding of the `endSession()` lifecycle contract
- Two-step back-button widget test is behaviorally complete: negative assertion on `'Session List'` before first back press, positive assertion after second ŌĆö would fail under the old `finally`-block behavior

- Unidirectional `session_providers ŌåÆ voice_providers` dependency maintained; Riverpod provider graph remains acyclic
