## Summary

Phase 8B adds local LLM integration via the ConversationLayer strategy pattern (ADR-0017): 6 new files (PersonalityConfig, LocalLlmService, LocalLlmLayer, LlmModelDownloadService, download dialog, personality providers) and 6 modified files (AgentRepository constructor injection, provider wiring, settings UI, pubspec). 822 tests pass at 80.7% coverage. The architecture is sound — constructor injection, session-locked layer policy, fallback chain, and custom prompt isolation all hold correctly. Two blocking findings (coding standard violations) must be fixed before merge; 18 advisory findings are recommended.

## Required Changes Before Merge

1. **Fix bare catch violations (3 sites)**: Change `catch (e)` to `on Exception catch (e)` in `lib/services/local_llm_service.dart:132`, change `catch (_)` to `on FormatException` in `lib/models/personality_config.dart:122`, change `catch (_)` to `on Exception` in `lib/ui/widgets/llm_model_download_dialog.dart:87`. Per coding_standards.md: "No bare catch — always catch specific exception types." [arch F5, sec F4, qa F8]

2. **Consolidate redundant catch blocks in AgentRepository**: Remove the `on ClaudeApiException` clause from all 4 public methods (`getGreeting`, `getFollowUp`, `generateSummary`, `getResumeGreeting`) — the `on Exception` clause already catches it. Add comment: `// Catches ClaudeApiException, LocalLlmException, and any layer failure — falls back to rule-based per ADR-0017 section 4`. [arch F3]

## Recommended Improvements (Non-Blocking)

1. **Document model load lifecycle gap**: The download-to-load wiring is Phase 8C scope per spec. Add a TODO comment in `llm_providers.dart` at `localLlmServiceProvider` noting this. [indep F1]
2. **Document Qwen/ChatML coupling in sanitizer**: Add a comment in `sanitizeCustomPrompt` noting the blocklist targets ChatML (Qwen's format) and must be reviewed on model swap. [sec F1]
3. **Add missing tests (4 medium-priority)**: localLlmLayerProvider conditionals, prefer-Claude+offline+LLM scenario, daysSinceLast boundary, dialog error/retry flow. [qa F1-F4]
4. **Consider wrapping custom prompt in labeled block**: `[Additional user instructions: $customPrompt]` in `effectiveSystemPrompt` to reduce injection surface. [sec F1]
5. **Expose modelFileName static getter**: Eliminate 3 sites of duplicated URI parsing. [arch F4]
6. **Consider mtime-based SHA-256 re-verification**: On model readiness check, compare file mtime to stored verified-mtime. [sec F2]
7. **Address STT model SHA-256 pre-existing debt**: Populate before production release. [sec F6]
