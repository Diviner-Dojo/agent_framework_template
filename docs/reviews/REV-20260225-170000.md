## Summary

Phase 12: Video Capture adds video recording, local playback, and Supabase sync infrastructure to the Agentic Journal app. The implementation creates a Videos drift table (schema v6), VideoService for capture/metadata-strip/thumbnail pipeline, video providers, a full-screen video player widget, and extends the media capture sheet and session screen. All 4 specialists identified a critical path resolution defect: relative paths stored in the database are passed directly to `File()` constructors in the UI, causing 100% failure rate for video thumbnails and playback. A second blocking finding is that SessionDetailScreen has no video rendering for past sessions.

## Findings by Specialist

### QA Specialist
- **HIGH**: `deleteVideoFile()` boundary check (path-traversal prevention) has zero test coverage
- **HIGH**: `attachVideo()` in SessionNotifier is completely untested — primary integration point
- **MEDIUM**: `calculateTotalSize()`, `VideoPlayerScreen`, ChatBubble video branch, and canonicalVideoPath assertions all lack adequate test coverage
- **LOW**: getVideosToSync all-synced edge case, deleteSessionVideos UUID guard, _makeRelativePath untested
- Confidence: 0.87

### Security Specialist
- **MEDIUM**: Relative paths passed to `VideoPlayerController.file()` and `Image.file()` without resolution against app support directory — feature fails silently
- **MEDIUM**: Stream-level GPS metadata on iOS `.mov` files survives `-map_metadata -1` — location data extractable from synced videos
- **LOW**: Missing UPDATE RLS policy on `storage.objects` in `004_video_schema.sql`
- **LOW**: Case-insensitive filesystem edge case in `deleteVideoFile` containment check
- **LOW**: `_makeRelativePath` fallback returns absolute path, creating inconsistent DB entries
- Confidence: 0.88

### Architecture Consultant
- **MEDIUM**: `_getBaseDirPath()` and `_makeRelativePath()` are domain logic placed in SessionNotifier — should be in VideoService per PhotoService pattern
- **MEDIUM**: `ProcessedVideo.durationSeconds` hardcoded to 0; `_handleVideoCapture` never extracts or passes duration — duration badge shows 0:00 for all videos
- **MEDIUM**: Asymmetric photo/video indexing in session screen uses dynamic casts — fragile if a message has both media types
- **LOW**: Inconsistent UUID generation (raw `Uuid()` for photos vs `generateUuid()` for videos)
- **LOW**: `deleteVideoFile` receives relative paths from DB but containment check expects absolute
- **LOW**: Video constants are public top-level but are implementation details
- Confidence: 0.88

### Independent Perspective
- **BLOCKING**: Relative path resolution failure affects 100% of captured videos — thumbnails show broken images, playback shows error state. Not an edge case; this is the primary code path.
- **BLOCKING**: `SessionDetailScreen` does not display videos — past sessions with video attachments show `[Video]` as plain text with no thumbnail or playback.
- **NON-BLOCKING**: `durationSeconds` permanently 0 — cosmetic but misleading
- **NON-BLOCKING**: `deleteVideoFile` broken for relative paths — dead code path
- **NON-BLOCKING**: Thumbnail failure silently discards entire video — user loses recording with no recovery
- Confidence: 0.82

## Required Changes Before Merge

1. **B1 — Resolve relative paths before File() construction**: All database-sourced `localPath` and `thumbnailPath` values must be resolved against `applicationSupportDirectory` before passing to `File()`, `Image.file()`, or `VideoPlayerController.file()`. Affects: `chat_bubble.dart`, `video_player_widget.dart`, and any other UI code that reads video paths from the database. (All 4 specialists flagged)

2. **B2 — Add video display to SessionDetailScreen**: The read-only session detail view must render video thumbnails with playback capability for past sessions, matching the live session experience. (independent-perspective)

## Recommended Improvements (Non-Blocking)

1. **A1**: Extract actual `durationSeconds` from video metadata via FFmpegKit probe or video_player initialization, pass to `attachVideo()`. (architecture-consultant, independent-perspective)
2. **A2**: Fix `deleteVideoFile` to resolve relative paths before containment check. (architecture-consultant, independent-perspective, security-specialist)
3. **A3**: Move `_getBaseDirPath()` and `_makeRelativePath()` from SessionNotifier to VideoService. (architecture-consultant)
4. **A4**: Add `-map_metadata:s -1` to FFmpegKit arguments for iOS stream-level GPS stripping, or document as known limitation in ADR-0021. (security-specialist)
5. **A5**: Add UPDATE policy on `storage.objects` in `004_video_schema.sql`. (security-specialist)
6. **A6**: Add unit tests for `attachVideo()` happy path, error handling, and state updates. (qa-specialist)
7. **A7**: Add unit tests for `deleteVideoFile()` boundary check. (qa-specialist)
8. **A8**: Add widget tests for `VideoPlayerScreen`. (qa-specialist)
9. **A9**: Add widget tests for ChatBubble video branch. (qa-specialist)
10. **A10**: On thumbnail failure, save video without thumbnail as fallback instead of discarding. (independent-perspective)
