## Summary

Phase 3C adds a spaced-repetition "Gift" card to the home screen, resurfacing past journal entries from ~7d/~30d/~90d windows. The service logic and 11-test suite are architecturally sound, ADHD-UX compliant, and well-documented. Five blocking gaps must be addressed before merge: three missing test paths in the service layer, one race condition in the Reflect button navigation, and one silent-failure bug in the Skip button.

## Request Context

- **What was requested**: Review Phase 3C Home Screen Resurfacing (Gift card) for the ship workflow review gate
- **Files/scope**: `lib/services/resurfacing_service.dart` (NEW), `lib/providers/resurfacing_providers.dart` (NEW), `lib/ui/screens/session_list_screen.dart` (modified — gift card added), `test/services/resurfacing_service_test.dart` (NEW — 11 tests)
- **Developer-stated motivation**: Phase 3C of ADHD roadmap — spaced-repetition resurfacing of past journal entries as 'Gift' cards on the home screen at ~7d/~30d/~90d windows (±3 days). Skip persists exclusion in SharedPreferences.
- **Explicit constraints**: ADHD UX clinical constraints — never reference missed days/gaps/absence duration, no streaks, no gap-shaming framing. Only one card at a time.

- **B-2 (Blocking)**: No test for incomplete session (endTime==null) eligibility path.
- **B-3 (Blocking)**: No test for empty-string summary eligibility path.
- **A-4**: `resurfacedSessionProvider` error state silently hides card — document as intentional.
- **A-5**: `_formatTimeAgo` boundary values untested (days 13, 14, 49, 50, 119, 120).

- **A-7**: Comment missing in skip test explaining determinism.

### Architecture Consultant
- **A-1**: `ref.read` in FutureProvider should be `ref.watch` for codebase consistency (one-word fix, zero behavioral impact).
- **A-2**: Unbounded SharedPreferences exclusion list — IDs for sessions >93d are dead weight. Add TODO comment.
- **A-3 (Info)**: Redundant null check on `summary` — simplifiable.
- **A-16 (Info)**: No ADR required for Phase 3C scope.

- ADHD clinical UX compliance confirmed.

### UX Evaluator
- **B-4 (Blocking)**: `ref.invalidate(resurfacedSessionProvider)` fires before `pushNamed` completes — race condition. Invalidate after await on Reflect button.
- **B-5 (Blocking)**: Skip button no error handling — if `skipSession()` throws, exclusion silently never persists but card disappears.
- **A-8**: Skip permanence not communicated — tooltip "Skip this memory" implies "not now" not "never again." Change to "Never resurface this memory."
- **A-9**: Card-level `Semantics` missing — screen readers land on card without context label.
- **A-10**: Cognitive load when banner and gift card both visible — similar color roles in dark mode.
- **A-11**: Decorative sparkle icon not excluded from semantics tree — wrap in `ExcludeSemantics`.
- **A-12**: Layout shift during provider reload — reserve vertical space during loading state.
- **A-13**: `_formatTimeAgo` "a while back" branch is dead code (max window 93 days) — add comment.
- ADHD clinical compliance: **PASSED** (all three specialists confirmed).
- Confidence: 0.88

## Required Changes Before Merge

1. **B-1** — Add widget tests for `_buildGiftCard`: card renders with qualifying session, Skip button calls `skipSession` and removes card, Reflect button navigates to `/session/detail` with correct `sessionId`.
2. **B-2** — Add service test: `'returns null for incomplete session (endTime is null)'` — `createSession()` without `endSession()` at 7 days, assert null.
3. **B-3** — Add service test: `'returns null when session summary is empty string'` — `endSession(id, startTime, summary: '')`, assert null.
4. **B-4** — Fix Reflect button: `await Navigator.of(context).pushNamed(...)` then `if (context.mounted) ref.invalidate(resurfacedSessionProvider)` after return.
5. **B-5** — Add try/catch in Skip button: only call `ref.invalidate` on success; show SnackBar `"Couldn't skip. Try again."` on failure.

## Recommended Improvements (Non-Blocking)

1. **A-1** — Change `ref.read(resurfacingServiceProvider)` to `ref.watch(resurfacingServiceProvider)` in `resurfacedSessionProvider` for codebase convention consistency.
2. **A-2** — Add `// TODO: prune excluded IDs for sessions older than max window (~93 days)` comment in `skipSession`.
3. **A-5** — Add widget test group for `_formatTimeAgo` boundary values via rendered label text.
4. **A-6** — Add service test: `'returns exactly one session when sessions exist in multiple windows'`.
5. **A-8** — Change skip tooltip to `'Never resurface this memory'` (communicates permanence without dialog interruption).
6. **A-9** — Add `Semantics` wrapper on gift card with descriptive label.
7. **A-11** — Wrap decorative `Icons.auto_awesome_outlined` in `ExcludeSemantics`.
8. **A-13** — Add comment to `_formatTimeAgo` noting branches beyond 120 days are unreachable given current window configuration.
9. **A-3** — Simplify redundant null check: replace `s.summary != null && (s.summary?.isNotEmpty ?? false)` with `s.summary?.isNotEmpty == true`.
