## Findings by Specialist

### QA Specialist

### UX Evaluator
- **[Advisory, Medium]** Success SnackBar displays full ISO-8601 filename (e.g. `agentic_journal_export_2026-03-05T12-30-00.000Z.json`). Long enough to overflow narrow phones. ADHD concern: truncated success message creates doubt. Suggest: `'Export saved to your Downloads folder.'` with optional `SnackBarAction('Open', ...)`.
- **[Advisory, Medium]** Error SnackBar renders raw Dart exception text (`Export failed: FileSystemException: ...`). No recovery guidance, no actionable next step. Error SnackBar also uses 4s default (shorter than success 6s — backwards for ADHD UX). Suggest: `FileSystemException` caught separately with plain-language message + 8s duration.
- **[Advisory, Low]** Loading state adequate for small datasets. Indeterminate spinner gives no proportional signal for large datasets (>3s). Non-blocking; no ADHD concern at current data sizes.
- **[Advisory, Low]** Doc comment says "video paths" but export includes full metadata objects. Internal-only, not user-facing.
- Confidence: 0.88

5. **Doc comment accuracy** (UX-A4): Update `_exportData()` doc comment — "photo paths, and video paths" → "photo metadata and video metadata (with dimensions and file size)".
