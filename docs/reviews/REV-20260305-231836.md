display bug fix: long USER messages in session_detail_screen and journal_session_screen

## Findings by Specialist

### UX Evaluator
- **BLOCKING → RESOLVED — UX-B1 [High]** — "Show more" toggle had no `Semantics` wrapper.
  TalkBack announced raw label with no button role, no hint, no association to truncated
  content. WCAG SC 4.1.2 violated. Fixed in-review: `Semantics(button: true, label: ...,
  hint: ...)` added.
- **BLOCKING → RESOLVED — UX-B2 [High]** — `GestureDetector` tap target ~15dp (labelSmall
  + 4dp padding). Below 48dp Material/WCAG 2.5.5 minimum. Fixed in-review: replaced with
  `InkWell` + `EdgeInsets.symmetric(vertical: 14)`.
- **UX-A1 [Medium]** — 300-char collapse threshold may trigger for messages that visually
  fit within 5 lines (false affordance). Recommend raising to 500 chars.
- **UX-A2 [Medium]** — Scroll recycling collapses `_isExpanded` state silently. Partially
  mitigated by `ValueKey` fix; full fix requires expansion state map in parent screen.
- **UX-A3 [Medium]** — FileSystemException SnackBar has no recovery action (correct copy,
  but user has no next step). Consider `SnackBarAction` opening storage settings.
- **UX-A4 [Low]** — Left axis Y-axis labels ("Low"/"Mid"/"High") not wrapped in
  `MediaQuery.noScaling`. Partial fix — only bottom axis covered. At 200% scale,
  "High" at 22sp clips in 36dp reservation.
- Confidence: 0.88

### QA Specialist
- **BLOCKING → RESOLVED — QA-B1 [High]** — `_CollapsibleMessageText` had no `key`.
  In `ListView.builder`, `_isExpanded` state could migrate to wrong message on list
  rebuild. Fixed in-review: `ValueKey(msg.messageId)` added to both call sites.
- **QA-A1 [Medium]** — No `didUpdateWidget` override (addressed in-review as part of
  blocking fix batch).
- **QA-A2 [Medium]** — No widget tests for `_CollapsibleMessageText`. Recommend 5 tests:
  short-no-toggle, long-shows-toggle, tap-expands, tap-collapses, assistant-never-collapses.
- **QA-A3 [Low]** — `isListening` test teardown ordering cosmetic inconsistency. No action.
- **QA-A4 [Low]** — `FileSystemException` error path has no test. Defer acceptable.
- Confidence: 0.88

## Recommended Improvements (Non-Blocking)

1. **Collapse threshold** (UX-A1): Raise `_kCollapseThreshold` from 300 to 500 chars
   to eliminate false-affordance zone where toggle appears but reveals no new content.
2. **_CollapsibleMessageText tests** (QA-A2): Add 5 widget tests to `chat_bubble_test.dart`
   covering threshold boundary, expand/collapse toggle, and assistant bypass.
3. **Left axis noScaling** (UX-A4): Apply `MediaQuery.noScaling` to leftTitles callbacks
   in both charts (same pattern as bottom axis fix).
4. **SnackBar recovery action** (UX-A3): Add `SnackBarAction` to FileSystemException
   SnackBar, or extend duration to 12s at minimum.
5. **Scroll state map** (UX-A2): Lift expansion state into a message-keyed map in the
   parent screen to survive scroll recycling.
