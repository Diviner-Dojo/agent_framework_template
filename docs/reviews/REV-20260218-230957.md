## Summary

Multi-agent review of the Todo API routes, database, and models. Four specialists examined the code across quality, security, architecture, and independent perspectives. Consensus on three required changes; independent perspective provided valuable calibration on context-appropriateness of security recommendations.

## Findings by Specialist

### QA Specialist (confidence: 0.92)
- 15 existing tests cover all 5 endpoints with good edge case coverage for validation
- Gaps: no parameterized boundary tests, no concurrent access tests, no pagination tests
- Whitespace-only titles pass validation but create unusable records
- Negative/zero todo_id not validated at path parameter level
- PATCH with empty body has undefined behavior

### Security Specialist (confidence: 0.85)
- **CRITICAL**: No authentication/authorization on any endpoint
- **HIGH**: No rate limiting or DoS protection
- **MEDIUM**: Input validation limited to length checks; no content validation
- **MEDIUM**: Database errors may leak stack traces
- **POSITIVE**: All SQL uses parameterized queries; no injection risk

### Architecture Consultant (confidence: 0.92)
- Global mutable state (`_db = None` / `set_database()`) is the critical antipattern
- Breaks testability, framework integration, and lifecycle safety
- Database class and Pydantic models are well-encapsulated
- Modern lifespan usage is good
- Single recommended change: migrate to `Depends()` pattern

### Independent Perspective (confidence: 0.78)
- Global state critique may be partially cargo-culting; real issue is connection pooling
- Auth recommendation assumes multi-user production use without evidence
- Hidden risk: SQLite concurrent writes with multiple Uvicorn workers
- Pre-mortem: uncontrolled DB growth degrades list endpoint within months
- Code may be exactly right for its context (learning/internal tool)

## Required Changes Before Merge

1. **Replace global `_db` with FastAPI `Depends()`** — Consensus across architecture, security, and testability. Addresses the structural coupling, enables test overrides, and prepares for per-request isolation.
2. **Add `Path(gt=0)` validation on `todo_id` parameters** — Negative and zero IDs are currently accepted. Trivial fix, real bug.
3. **Add whitespace stripping/validation on title** — Whitespace-only titles create unusable records. Add `strip_whitespace=True` or custom validator.

## Recommended Improvements (Non-Blocking)

1. Add pagination to GET /todos (limit/offset parameters)
2. Add CORS middleware with explicit allowed origins
3. Add global exception handler to prevent stack trace leakage
4. Document auth and rate-limiting decisions as ADRs (defer vs. implement)

- **Bloom's levels**: Understand (explain Depends flow), Apply (refactor the global to use it)
