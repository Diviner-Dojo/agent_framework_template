## Required Changes Before Merge

1. **Fix FK violation in unify_sightings.py**: Synthetic discussion_ids (`adoption-log-<source>`) don't exist in the `discussions` table. Either make `pattern_sightings.discussion_id` nullable for adoption-log source rows, or register synthetic discussion rows before inserting sightings.

2. **Unify _pattern_hash tokenization**: `mine_patterns.py` uses `_tokenize()` which filters words with `len > 2`, while `unify_sightings.py` uses inline `re.findall` without the length filter. The same summary produces different hashes in the two modules, breaking cross-source Rule of Three counting. Extract both into a shared utility or make the implementations identical.
