## Summary

Phase 7B: Continuous Voice Mode adds a state-machine orchestrator (`VoiceSessionOrchestrator`) for the continuous listen‚Üíprocess‚Üíspeak‚Üílisten loop, voice command detection (end/discard/undo), error recovery with verbal announcements, auto-save on backgrounding, and Android assistant-gesture voice launch. The review covered 10 files (3 new, 7 modified) across services, providers, UI, and platform code. All 4 specialists converge at 0.87 confidence that the architecture is sound with specific correctness and safety fixes needed.

## Findings by Specialist

### Architecture Consultant
- **[Blocking]** Unawaited future in `_speakNonBlocking` (line 783) ‚Äî fire-and-forget TTS call has no `.catchError()`, producing unhandled async exceptions in release mode
- **[Blocking]** ADR gap ‚Äî Phase 7B architectural decisions (callback pattern, ValueNotifier, classifier separation, sentence-splitting TTS) not captured in any ADR
- **[Advisory]** Mutable callback fields allow invalid orchestrator state between construction and wiring
- **[Advisory]** ValueNotifier diverges from Riverpod pattern (acceptable trade-off, needs ADR documentation)
- **[Advisory]** VoiceCommandClassifier created inline rather than injected via constructor
- **[Advisory]** Bare `catch(e)` instead of `on Exception catch(e)` in several locations
- Confidence: 0.87

### QA Specialist
- **[Blocking]** Race condition: `_executeEndSession`/`_executeDiscard` don't re-check `state.phase` after awaited `_speak()` calls ‚Äî `stop()` during TTS confirmation still executes destructive action
- **[Blocking]** Classifier test assertion wrong (line 110) ‚Äî expects `VoiceCommand.none` for "I'm done with the dishes" but classifier returns `endSession` at confidence 0.5
- **[Advisory]** `resumeLatestSession()` has zero test coverage
- **[Advisory]** `onAssistantMessage()` when phase is NOT processing is untested
- **[Advisory]** Undo timer expiry path untested
- **[Advisory]** `_isAffirmative()` never tested in isolation
- **[Advisory]** App lifecycle auto-save/pause/resume has no widget tests
- Confidence: 0.87

### UX Evaluator
- **[Advisory]** Error phase indicator never renders ‚Äî `_handleError` sets `isContinuousMode: false` before UI can show error chip
- **[Advisory]** Auto-save defaults to `true` without user consent on first install
- **[Advisory]** Error recovery drops continuous mode permanently ‚Äî no automatic retry
- **[Advisory]** Phase indicator lacks `Semantics`/`liveRegion` for accessibility
- **[Advisory]** Long-press for continuous mode is a hidden gesture with no coach mark
- **[Advisory]** Undo window: TTS says "say undo" but STT is off ‚Äî contradicts hands-free promise
- **[Advisory]** Processing phase has no visual feedback beyond 12sp "Thinking" chip
- Confidence: 0.87

### Security Specialist
- **[Blocking]** Unbounded confirmation timeout ‚Äî `_awaitingConfirmation` has no timeout, allowing ambient audio (TV saying "yes") to trigger destructive discard after indefinite window
- **[Blocking]** "stop"/"finish"/"bye" in `_strongEndPattern` ‚Äî too common in narrative speech for high-confidence end command (executes without confirmation)
- **[Blocking]** Stale confirmation state on dispose ‚Äî `_awaitingConfirmation` and `_pendingCommand` not reset in `dispose()` or `startContinuousMode()`
- Platform channel flags (VOICE_ASSIST intent) correctly implemented with one-shot clear pattern
- Confidence: 0.87

## Required Changes Before Merge

1. **[arch-7]** Add `.catchError()` to unawaited TTS future in `_speakNonBlocking` to prevent unhandled async exceptions in release
2. **[qa-3]** Add phase guard after each `await _speak()` in `_executeEndSession`, `_executeDiscard`, and `_requestConfirmation` ‚Äî if `state.phase` changed during await, abort the action
3. **[security-1]** Add 10-second bounded timeout on `_awaitingConfirmation` ‚Äî reset to false and cancel pending command on expiry
4. **[security-2]** Move "stop", "finish", "bye" from `_strongEndPattern` to `_moderateEndPattern` (requires verbal confirmation instead of direct execution)
5. **[security-3]** Reset `_awaitingConfirmation` and `_pendingCommand` in `dispose()`, `startContinuousMode()`, and `startPushToTalk()`
6. **[qa-6]** Fix classifier test assertion on line 110 ‚Äî change from `expect(result.command, VoiceCommand.none)` to `expect(result.command, VoiceCommand.endSession)` with `expect(result.confidence, lessThan(0.8))`
