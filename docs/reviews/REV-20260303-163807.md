## Summary

Three post-deploy bug fixes reviewed: STT endpointing increased from 2s to 3s (ADR-0031 alignment), stale check-in state cleared via `cancelCheckIn()` else branch in `_maybeStartCheckIn()`, and data export redirected to public Downloads folder per developer request. Two blocking findings identified: the regression test for the check-in fix does not pre-seed `isActive=true` (making it a weak assertion), and the Downloads path change lacks a SnackBar test. Security raised an advisory concern about world-readable public storage for personal journal data, treated as advisory per developer's explicit design intent.

## Request Context

- **What was requested**: Fix 3 post-deploy bugs reported after v0.18.0+13 deploy to SM_G998U1: (1) STT cuts off after one word, (2) new journal entry shows check-in card instead of text input after voice mode, (3) export file not accessible from Files app
- **Files/scope**: lib/services/speech_to_text_stt_service.dart, lib/ui/screens/journal_session_screen.dart, lib/ui/screens/settings_screen.dart, integration_test/smoke_test.dart, test/ui/journal_session_screen_test.dart
- **Developer-stated motivation**: Physical device testing revealed 3 regressions after Phase 1 Pulse Check-In / v0.18.0+13 deploy
- **Explicit constraints**: Downloads folder export is developer-requested; security concerns for this feature are advisory, not blocking

### UX Evaluator
- **[MEDIUM — advisory]** STT 3s silence gap has no visual state change; user sees static 'Listening...' with no 'receipt' signal after speech ends
- **[LOW — advisory]** Check-in complete card has no 'Continue journaling' button — ADHD users may not know how to proceed
- **[LOW — advisory]** SnackBar with full filename may overflow on 360dp devices
- **[INFO]** No ADHD UX constraint violations
- Confidence: 0.87

### Security Specialist
- **[HIGH — advisory]** Personal journal data (transcripts, mood scores) in world-readable `/storage/emulated/0/Download` — MediaStore API recommended for scoped access. Advisory per developer's explicit feature request.
- **[MEDIUM — advisory]** Hardcoded Downloads path unreliable on OEM Android variants; use `getDownloadsDirectory()` from path_provider
- **[LOW — advisory]** Raw exception message in SnackBar catch — surface generic 'Export failed' message
- **[LOW — advisory]** READ_EXTERNAL_STORAGE permission review needed for API 33+ targets
- Confidence: 0.88

## Required Changes Before Merge

1. **[B1]** Pre-seed `checkInProvider` with `isActive=true` before widget build in the regression test (`test/ui/journal_session_screen_test.dart`) so the test proves the fix prevents the regression, not just that the initial default state is correct.

2. **[B2]** Add a test to `test/ui/settings_screen_expanded_test.dart` (or equivalent) asserting the 'Saved to Downloads:' SnackBar text appears when the export button is tapped.

## Recommended Improvements (Non-Blocking)

1. **[A1]** Use MediaStore API instead of hardcoded path for Downloads export (security, OEM compatibility). Document the chosen tradeoff via inline comment or ADR.
2. **[A2]** Replace hardcoded `/storage/emulated/0/Download` with `getDownloadsDirectory()` from path_provider (resolves Security-M and UX-L simultaneously).
3. **[A3]** After STT `isFinal=true` event, update status label to 'Got it...' or similar to reduce ADHD-relevant UI ambiguity during 3s silence window. Deferred to Deepgram P1 sprint.
4. **[A4]** Add 'Continue journaling' affordance to PulseCheckInSummary card.
5. **[A5]** Shorten export SnackBar to 'Journal data saved to Downloads.' (resolved by B2 fix).
6. **[A6]** Surface generic 'Export failed. Check storage permissions.' in catch blocks.
7. **[A7]** Extract STT pauseFor rationale comment to class-level doc comment.
