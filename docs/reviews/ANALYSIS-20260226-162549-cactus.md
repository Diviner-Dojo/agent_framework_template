## Project Profile

- **Name**: Cactus (cactus-compute/cactus)
- **Source**: https://github.com/cactus-compute/cactus
- **Tech Stack**: C++ (llama.cpp fork), Dart/Flutter SDK, React Native SDK, TypeScript
- **Domain**: On-device LLM inference, mobile-optimized
- **Maturity**: Active development, custom ARM optimization, moving toward proprietary model format

## Synthesis

4 patterns identified. ADOPT: ARM flag insight (`armv8.2-a+dotprod+fp16` fixes SIGILL), CompletionResult telemetry fields. DEFER: concurrent local+cloud handoff (concept, not implementation). AVOID: full library adoption (SSL-off-by-default, unconsented telemetry, proprietary .cact format, custom license).

Key finding: the SIGILL fix is a **build flag**, not a library swap.

## Pattern Recommendations

### ADOPT

| Pattern | Score | Enhancement | Priority |
|---------|-------|-------------|----------|
| ARM build flag fix | 22/25 | E3 | P0 |
| CompletionResult telemetry fields | 20/25 | E28 | P3 |

**ARM Build Flag Fix**: Change llamadart's ARM build flag from `-march=armv8.7-a` to `-march=armv8.2-a+dotprod+fp16`. Snapdragon 888 (SM8350) implements ARMv8.2-A with dotprod but NOT ARMv8.7-A. The armv8.7-a flag emits instructions the Cortex-A78 cannot execute. Cactus ships on armv8.2-a baseline. llama.cpp GitHub issue #12393 confirms root cause.

**CompletionResult Telemetry**: Add `prefillTps`, `decodeTps`, `timeToFirstToken`, `confidence` fields to on-device LLM response type. Makes inference quality observable without separate instrumentation.

### DEFER

| Pattern | Score | Enhancement | Priority |
|---------|-------|-------------|----------|
| Concurrent local+cloud handoff | 18/25 | E27 | P4 |

**Concurrent Handoff**: When local LLM confidence drops, fire cloud request concurrently. Concept only — Cactus doesn't fully implement this. Implementation would be in our Dart/Riverpod layer, not via Cactus library.

### AVOID

**Full Cactus Library Adoption**: Multiple red flags:
- SSL verification disabled by default (security risk)
- Unconsented analytics telemetry (privacy concern)
- Proprietary `.cact` model format (vendor lock-in)
- Custom license with $2M ARR limit and auto-termination clause
- The useful insight (ARM build flags) is a build config change, not a library dependency

## License Impact

Custom source-available license — **AVOID full adoption**. The ARM build flag insight and telemetry field pattern are generic knowledge extracted from examining the project, not code that requires licensing. No Cactus code should be incorporated.
