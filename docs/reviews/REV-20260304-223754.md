## Request Context

- **What was requested**: Fix intent classifier natural language flexibility — 'I need to set a calendar item for Friday night with Shawn' was routed as journal instead of calendarEvent. Add support for bare weekday+time-of-day temporal references ('friday night', 'saturday morning').
- **Files/scope**: `lib/services/intent_classifier.dart`, `test/services/intent_classifier_test.dart`, `memory/bugs/regression-ledger.md`
- **Developer-stated motivation**: Device testing revealed natural language phrases like 'I need to set a calendar item' and 'for Friday night' were not recognized as scheduling intent. User wants more permissive/flexible NL recognition.
- **Explicit constraints**: No false positives on journaling sentences. Regression tests required.

## Verdict: approve-with-changes

1 blocking finding resolved in-review. 3 advisory findings remain open.

## Blocking Finding Resolved In-Review

### B1 — `\bcalendar\s+(item|entry)\b` too broad without verb guard ✅ resolved
- **Severity**: High
- **Specialists**: qa-specialist + independent-perspective (consensus)
- **Original risk**: The pattern `\bcalendar\s+(item|entry)\b` matched ANY sentence containing the phrase — including past-narrative contexts like "I remember we had a calendar item last week" (scored calendarEvent at 0.75 confidence via compound-noun hit + temporal boost).
- **Fix**: Replaced bare compound-noun match with a two-arm pattern requiring either a scheduling-verb within 40 characters (arm 1: `(schedule|book|set up|add|create|plan|set|make|put).{0,40}calendar (item|entry)`) OR a future-temporal preposition following (arm 2: `calendar (item|entry).{0,40}(for|on|this|next|tomorrow)`).
- **Verification**: Added false-positive guard tests for "I remember we had a calendar item last week" and "The calendar entry got deleted" — both correctly route to `journal`. Added positive test for "I need to set a calendar entry for next Monday" — correctly routes to `calendarEvent`.

## Advisory Findings (open)

### A1 — `on (weekday)` in `_temporalPattern` temporal-boost branch untested (Low)
- **Specialist**: qa-specialist
- `on (weekday)` was already in `_futureTemporalPattern` but is newly added to `_temporalPattern`. The temporal-boost branch at line 246 fires when `_temporalPattern` matches and `calendarScore > 0`. No test exercises this combined path (as distinct from the `_futureTemporalPattern` eventNoun guard). Advisory: add `'schedule a meeting on Monday'` to confirm 0.75 combined confidence from `^(schedule)\b` (+0.5) plus temporal boost (+0.25).

### A2 — `'I was so nervous about the meeting on Friday'` behavior undocumented (Low)
- **Specialist**: qa-specialist + independent-perspective
- Event noun ("meeting") + `on friday` in `_futureTemporalPattern` scores +0.4 calendar via the eventNoun+futureTemporal guard. No verb guard prevents this. This is likely an accepted false positive given the confirmation gate (ADR-0020 §8), but it should be documented as intentional or tested.

### A3 — Confirmation gate is load-bearing for ADHD UX (Low/Architecture)
- **Specialist**: independent-perspective
- As the intent classifier grows more permissive, the UX disruption risk increases for active journaling users who discuss past events. A journaling user unexpectedly seeing "Schedule this event?" while narrating violates the spirit of the ADHD clinical UX constraints in CLAUDE.md. Advisory: consider tracking calendar-prompt dismissal rate to detect false-positive drift over time.

The temporal additions are correctly scoped — `_futureTemporalPattern` is only used at the eventNoun+futureTemporal guard (line 178–180), preventing past-temporal false positives from that path. The `_hasFutureActionContext` addition is bounded by the modal prefix requirement ("need to," "want to," etc.).
